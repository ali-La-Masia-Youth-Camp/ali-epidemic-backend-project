## git提交参考规范
1. 个人新建单独分支进行开发
2. 每次提交前先拉取一遍，解决完冲突再提交
3. 提交规范
    1. commit格式为：git commit -m "[type]: [info]"，type可以分为下面几类
        - add：提交新功能
        - fix：修改bug
        - docs：只修改了文档
        - style：调整代码格式，没有修改代码逻辑（例如修改 tslint 提出的代码样式的问题）
